# オセロAI学習ロードマップ（新要件版）

## 1. 目的
- 最終目標: 一般的なプレイヤーに安定して勝てるオセロAIを作る。
- 中間目標: C++で対局データを生成し、Pythonで学習し、学習済み成果を再利用して次のデータ生成に反映するループを作る。

## 2. 現状認識
- 現在の学習は主に局面価値予測（勝敗方向の推定）が中心。
- 本計画では Value only を採用し、方策分布（Policy head）は導入しない。
- 着手は「合法手を列挙 → 各手の1手先局面を価値評価 → 最大値の手を選択」で決定する。
- 連携形式は単一バイナリファイル `ai/data/dataset.bin` を標準とする。

## 3. データ仕様（共通）
- 1局面の固定長レコード（19バイト）
    - `uint64 black`（8バイト）
    - `uint64 white`（8バイト）
    - `int8 black_to_move`（1バイト）
    - `int8 move`（1バイト）
    - `int8 result`（1バイト）
- Python側は `np.fromfile` + 構造化dtypeでパースする。
- 盤面入力は8x8の3チャネル（黒/白/手番）を標準とする。

## 4. 段階別計画

### Phase 2: 価値学習の土台を固める
- 作るデータ
    - ランダム自己対局から得た局面 + 最終勝敗ラベル
- 予測対象
    - 局面価値（-1, 0, 1）
- 推論利用
    - 局面評価器として利用（まずは学習安定化を優先）
- 完了条件
    - 学習が安定して完走
    - `val_loss` と符号一致率を継続監視
    - wandbで比較可能な履歴が残る

### Phase 3: 精度改善と重み再利用
- 作るデータ
    - 既存データ + 新規自己対局データ
    - 先手後手、勝敗、手数帯の偏りを確認したデータ
- 予測対象
    - 局面価値（継続）
- 推論利用
    - 既存重みを初期値として再学習（モデル重みの再利用）
    - 実験比較で有効な設定を確定
- 完了条件
    - 重み再利用ありで再現可能に学習できる
    - 3チャネル入力で学習が安定し、指標が再現可能
    - 主要指標が基準runより改善

### Phase 4: C++自己対局へPython学習成果を反映
- 作るデータ
    - Python学習結果（重み）を使って生成した自己対局データ
- 予測対象
    - 局面価値（Value only）
- 推論利用
    - C++自己対局を完全ランダムから脱却し、毎手で合法手のみを列挙する
    - 各合法手をAIで評価し、最も評価値が高い手（最善手）を選択する
    - 同点時のみ固定ルールまたは乱択でタイブレークする
- 完了条件
    - C++側がPython側の学習成果を読み取り可能
    - 自己対局が「AIが選択した合法手のみ」で最後まで進行する
    - その対局から新規 `dataset.bin` を生成可能
    - ランダム方式より対局品質が改善

### Phase 5: 自動学習ループの確立
- 作るデータ
    - 最新モデルが最善手選択で生成した自己対局データを継続追記
    - 過去データを混ぜるリプレイバッファ
- 予測対象
    - 価値予測（最善手選択のための評価値）
- 推論利用
    - 生成→学習→評価→採用を反復
- 完了条件
    - 旧新モデル対戦で改善傾向を維持
    - 対人想定評価で目標勝率に到達

## 5. モデル重み再利用方針
- 学習ごとに重みを保存し、次回学習で再開する。
- 可能ならoptimizer状態も保存し、途中再開の再現性を高める。
- モデルIDとデータ生成元の対応を記録し、性能差の原因を追跡可能にする。

## 6. C++とPythonの接続方針
- Python側はC++が読める形式で学習成果を出力する。
- C++側はその成果を使って、合法手集合の中から「1手先価値が最大の手」を毎手選択する。
- これにより、データ生成自体の品質を段階的に向上させる。

## 7. 直近アクション
1. Phase 3の評価軸を固定する（`val_loss`、符号一致率、比較基準run）。
2. 重み再利用の運用ルールを固定する（保存名、再開手順、wandb連携）。
3. C++に渡す学習成果の形式を固定する（重みと推論I/F）。
4. ランダム対局との差分評価を定期的に実施する。
