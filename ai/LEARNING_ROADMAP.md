# オセロAI自作プロジェクト：学習・実装ロードマップ（現仕様）

## 1. 目的
- 最終目標: 強化学習で「自分に勝てる」オセロAIを作る。
- MVP目標: C++で自己対戦データを生成し、Python（PyTorch）で学習できる状態にする。

## 2. 現状（2026-03時点）
- Phase 1（C++エンジンMVP）は完了。
- C++側で自己対戦データをCSV/バイナリ出力できる。
- 今後の主要受け渡し形式は単一バイナリファイル `ai/data/dataset.bin` とする。
- Phase 2（Python側MVP）は未着手。

## 3. 連携方針（今回の仕様）
- MVPでは「ファイル連携」を採用する。
- 数万局以上のデータを扱うため、受け渡しは単一バイナリファイルを優先する。
- CSVはデバッグ用の補助出力としてのみ利用する。
- Python側は `np.fromfile` で一括ロードする。

### 3.1 バイナリレコード仕様（固定長）
- C++/Pythonで同一仕様を共有するため、1局面を以下の固定長レコードで扱う。
  - `uint64 black`（8バイト）
  - `uint64 white`（8バイト）
  - `int8 black_to_move`（1バイト, 1:黒手番 / 0:白手番）
  - `int8 move`（1バイト, 0-63 / パスは-1）
  - `int8 result`（1バイト, 1:黒勝 / 0:引分 / -1:白勝）
- 総サイズは 19 バイト/レコードとし、構造体パディングに依存しないように逐次書き込みで保存する。
- 書き込み方式は追記（append）を基本とし、1つの `dataset.bin` に蓄積する。

## 4. 実装フェーズ

### Phase 2: Python側MVP（今ここ）
1. `ai/python/dataset.py` を実装する。
    - `ai/data/dataset.bin` を読み込み、学習用サンプルに変換する。
    - `np.fromfile` + `np.dtype`（構造化配列）で固定長レコードを安全にパースする。
    - 入力: `black`, `white`, `black_to_move` から盤面テンソルを作る。
    - 目標ラベル: `result`（黒勝=1, 引分=0, 白勝=-1）を利用する。
2. 盤面前処理を実装する。
    - `uint64`ビットボードを 8x8 の2チャネル（黒石/白石）へ変換する。
    - 必要に応じて手番チャネル（`black_to_move`）を追加し、2 or 3チャネルを選べる設計にする。
3. `ai/python/train.py` を実装する。
    - 小さなCNNまたはMLPを定義する（まずは学習が回る最小構成）。
    - DataLoaderで学習ループを実装する（損失はMSEで開始）。
    - Weights & Biases（wandb）で `train_loss`, `val_loss`, `epoch`, `learning_rate` を記録する。
    - `memo_wandb.py` と同様に `wandb.init()` / `wandb.log()` / `wandb.finish()` を利用する。
4. 動作確認を行う。
    - Datasetの長さ、テンソルshape、ラベル分布を確認する。
    - 1エポック以上の学習が正常終了することを確認する。
    - wandbダッシュボード上でloss推移が確認できることを確認する。

### Phase 3: 精度改善・学習サイクル準備
1. 精度改善（何を変えるか）
    - データ面: 局面の偏りを減らす（先手/後手、勝敗、手数帯のバランスを確認）。
    - 入力面: 2チャネル（黒/白）と3チャネル（+手番）を比較する。
    - モデル面: 小型CNNから段階的に層数・チャネル数を増やし、過学習を監視する。
    - 学習面: 学習率、バッチサイズ、重み減衰を探索する。
2. どう評価するか
    - 学習データと検証データを分離し、`val_loss` を主指標にする。
    - 追加指標として、勝敗符号の一致率（`sign(pred) == result`）を記録する。
    - wandbで実験条件（ハイパーパラメータ）と結果をセットで管理する。
3. サイクル準備
    - 「どのモデルで生成した対局データか」を記録するため、データ生成時にモデルIDを管理する。
    - 学習済みモデルを定期保存し、次フェーズの自己対戦生成に渡せる形にする。

### Phase 4: C++/Python統合強化（将来）
1. 学習済み重みの利用方式を決定（ONNX/推論ラッパーなど）。
2. C++自己対戦をランダム手から方策選択へ移行（例: $\epsilon$-greedy）。
3. 学習サイクルを自動化する。
    - Step A: 現モデルで自己対戦し、`dataset.bin` に追記。
    - Step B: 新データを含めて再学習し、新モデルを保存。
    - Step C: 旧モデルと新モデルを対戦評価し、採用モデルを更新。
    - Step D: Aに戻る。
4. 退化防止として、過去データの一部を混ぜるリプレイバッファ方式を導入する。

## 5. スコープ外（MVPではやらない）
- いきなり高度な強化学習アルゴリズム（MCTS併用や大規模分散学習）。
- pybind11による直接連携の最適化（必要になってから着手）。

## 6. 直近の実装順（小さく進める）
1. `dataset.py`のみ先に作成して `dataset.bin` 読み込み検証。
2. 承認後、`train.py`を作成して最小学習ループ + wandbログ出力を実装。
3. 承認後、`tasks.md`を更新してPhase 2完了条件を順に満たす。
